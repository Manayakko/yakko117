Code_Easy_Audit

首先看源码吧

php

这题考几个点：

  1. $_REQUEST 的变量覆盖
  2. 编码绕过
  3. PHP数组特性
  4. 正则绕过
  5. file_get_contents 函数

具体分析

php

上述代码把 $_GET, $_POST, $_COOKIE传进来的值进行了正则匹配，如果包含了A-Z 和 a-z的字母就被waf

这里的 $_REQUEST有一个特性，就是当 GET 和 POST 都存在同一个变量名的时候，只获取 POST 中的值，所以可以通过这个特性来绕过正则的匹配

当POST中也有个变量的时候，abc就会输出，否则就输出waf

接着往下分析

php

关于 $_SERVER['QUERY_STRING'] 获取的值，就是get中变量名和内容

详细资料可以参考一篇CSDN详解
这个绕过思路就是 URL编码

然后就是下面的代码

php

一个简单的 MD5 绕过
if(!(substr($_GET['cyber'], 32) === md5($_GET['cyber'])))
这里的绕过思路很简单，传进去是个数组的时候，值就为空，自然就相等了

然后是
if(preg_match('/^ciscnsec$/', $_GET['ciscn']) && $_GET['ciscn'] !== 'ciscnsec')
preg_match()函数没有规定开头，ciscn=ciscnsec （URL加密）

file_get_contents 函数
if(isset($getflag) && $getflag === 'security')

flag=data://text/plain,security
